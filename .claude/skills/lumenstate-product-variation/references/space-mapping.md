# 인테리어 공간 매핑 (`product.type` → Setting)

`scripts/generate.mjs` 의 `SPACE_BY_TYPE` 와 짝을 이룬다. `PRODUCT_TYPES`(products.js) 토큰을 키로 쓴다. 레퍼런스 제품을 **실제 설치된 모습**으로 배치할 공간 + 설치 방식을 정의한다.

**컷마다 배경이 달라야 한다.** 그래서 각 타입은 **5개의 서로 다른 배경 변형**(배열)을 가지며, 컷 레시피의 `bg` 인덱스로 컷마다 다른 배경을 고른다. 각 문자열은 `the fixture is ___` 뒤에 자연스럽게 이어지도록 서술한다.

| type 토큰 | 배경 변형 (bg 0 → 4) |
|---|---|
| `CEILING` | 거실·다이닝 천장 / 오픈 키친-다이닝 / 현관 포이어 / 침실 / **거실+소파(인물)** |
| `STAND` | 거실 코너+소파 / 창가 리딩누크 / 침실 코너 / 로프트 라운지 / **서재 코너(인물)** |
| `WALL` | 복도 벽+콘솔 / 거실 사이드보드 위 / 계단실 / 파우더룸 거울 옆 / **침실 리딩코너(인물)** |
| `DESK` | 월넛 책상 / 침대 옆 사이드테이블 / 복도 콘솔 / 키친 카운터 / **스튜디오 데스크(인물)** |

> **index 4 는 `lifestyle-person` 전용 — 사람이 머무는 거주 공간**(거실·침실 리딩코너 등)으로 둔다. 파우더룸·계단실·현관 등 사람이 책 읽기 어색한 곳은 인물 컷에 쓰지 않는다.

## 설치 고정 (`INSTALL_BY_TYPE`)

평면 마운트 타입은 공중부양·형태 재해석이 자주 발생한다. 타입별로 설치면 밀착을 강제한다:

| type | 설치 강제 |
|---|---|
| `CEILING` | 천장에 flush, 공중부양 금지, 얇은 원형 프로파일·실루엣 유지(드럼 금지) |
| `WALL` | 벽면에 flat·flush, 공중부양 금지, **정면에 가깝게**(실루엣·대칭 유지), 비대칭/독립형 3D로 재해석 금지 |
| `STAND` / `DESK` | 바닥/면 위에 레퍼런스대로 |

## 원칙

- **제품은 레퍼런스로 100% 고정**, 공간은 그 제품이 자연스럽게 놓이는 무대다. 공간이 제품을 압도하지 않는다(03-visual-direction "빛이 주인공").
- 가구·소품은 최소한, wall-tint 페이퍼 톤·웜 뉴트럴·넉넉한 여백.
- **컷마다 배경·구도를 크게 바꾼다** — 와이드 establishing, 미디엄, 매크로 클로즈(얕은 심도), 로우앵글, 인물 등장 라이프스타일까지 대비를 크게. 단, 제품 자체는 모든 컷에서 레퍼런스와 동일하다.
- `lifestyle-person` 컷은 배경에 인물을 캔디드로 추가한다(카메라 응시 X).

> 매핑 추가/수정 시: products.js 의 `PRODUCT_TYPES` 에 새 타입이 생기면 여기와 `SPACE_BY_TYPE`(배열) 양쪽에 추가한다. 미정의 타입은 스크립트에서 `DEFAULT_BACKGROUNDS` 로 폴백한다.
