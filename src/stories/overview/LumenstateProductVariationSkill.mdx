# Lumenstate — Product Variation Skill

> 제품 상세페이지의 **베리에이션 컷(메인 이미지 아래 갤러리 썸네일)** 을 일관된 브랜드 톤으로 자동 생성하는 도메인 한정 스킬.
> 위치: `.claude/skills/lumenstate-product-variation/`

---

## 한 줄 정의

`src/data/products.js` 의 제품 1종 + 재료 옵션을 받아, **동일 제품**을 **각도 · 줌 · 디테일 크롭**으로 변주하고 그 변주를 **빛의 연속성(주간 → 야간 점등)** 타임라인 위에 배치해 N장을 생성한다. 모든 컷은 **동일 카메라 · 동일 브랜드 톤 · 고정 재료**를 공유한다. 렌더는 Gemini(Nano Banana 2) 이미지 모델이 담당한다.

> 메인 hero 컷(`src/assets/product/{id}.png`)은 이미 존재하므로 **이 스킬은 hero를 만들지 않는다.** 갤러리에 깔리는 보조 컷만 생성한다.

---

## 언제 발동하는가

사용자가 **Lumenstate 제품 베리에이션 / 상세 갤러리 컷 / 베리에이션 썸네일 생성**을 말하거나, products.js 의 제품을 브랜드 톤 이미지로 만들어 달라고 할 때. `Lumenstate`, `베리에이션`, `제품 컷`, `Nano Banana`, `gemini image` 키워드가 트리거. **Lumenstate 도메인 전용**이며 임의 이미지 생성·일반 목업과 구분된다.

## 6 설계 기준

1. **톤** — warm minimal, ambient warmth (03 Visual Direction 의 flat & quiet 계승)
2. **베리에이션 축** — 컷 종류(각도·줌) / 점등 상태 / 공간 컨텍스트
3. **베이스** — eye-level 3/4 view, rule-of-thirds (카메라 고정)
4. **조명** — `kelvin`→색온도, `lux`→광량, `lightPattern`→빛 확산 형태 시각화
5. **공간** — `product.type` 자동 매핑 (CEILING→거실/다이닝, STAND→거실 코너, WALL→복도/침실 벽, DESK→책상)
6. **템플릿** — 5요소 `[Style][Subject][Setting][Action][Composition]`

## 5요소 템플릿

```
[Style]       모든 컷 고정. 브랜드 톤(warm minimal, flat & quiet, 3800K 웜 뉴트럴).
[Subject]     form + glassFinish + hardware. 호출 내 고정.
[Setting]     product.type → 공간 컨텍스트.
[Action]      kelvin + lux + lightPattern + 시간대. 컷마다 변동.
[Composition] 컷 각도·줌 + 고정 카메라(50mm, eye-level, 1:1, rule-of-thirds).
```

### 조립 예 (id 17 Lumen Dome, opaline + patina-brass, `material-macro`)

```
[Subject] A hemisphere desk lamp finished in opaline (milky white) glass with aged
          patina brass hardware — fixed across every cut.
[Action]  on, soft specular highlights on the hardware, deep candle-warm amber
          (~2400-2700K), even glow radiating in every direction. Early evening.
[Composition] macro detail crop where glass meets hardware, tight close-up. 1:1.
```

### 조명 (`kelvin` / `lux` / `lightPattern`)

- **kelvin** → 색온도 서술 (≤2700K 캔들 엠버 … >4200K 선명한 중성백). 4400K 라도 "차가운 백색"이 아닌 "선명한 중성백".
- **lux** → 광량 (≤180 intimate · ≤300 comfortable · >300 confident).
- **lightPattern** → 빛이 퍼지는 형태(downward/halo/omnidirectional/gap/refraction…)를 자연어 시각화로. **글로우 블룸·렌즈 플레어·방향성 드롭섀도는 만들지 않는다**(flat & quiet).

## Gemini API · 환경 설정

- 모델: `gemini-3.1-flash-image-preview` (Nano Banana 2)
- 패키지: `@google/genai` — `ai.models.generateContent({ model, contents, config })`
- `config.imageConfig`: `aspectRatio: '1:1'`, `imageSize: '1K'`
  - `1024px` 요구는 SDK 값 `'1K'` 에 대응(`1K/2K/4K`). 현재 이 프리뷰 모델은 `imageSize` 를 무시하고 1K 를 반환하는 알려진 이슈(js-genai #1461)가 있으나, 1K ≈ 1024² 라 실용상 무방.
- 반환: `response.candidates[0].content.parts[].inlineData.data` (base64) → 파일 저장.

### API 키 가이드

1. [aistudio.google.com](https://aistudio.google.com) 에서 키 발급
2. 프로젝트 루트 `.env.local` 에 `GEMINI_API_KEY=발급키`
3. `.gitignore` 의 `*.local` 규칙이 `.env.local` 을 제외 (이미 적용됨, 확인만)
4. `process.env.GEMINI_API_KEY` 참조. **미설정 시 명확한 에러로 즉시 종료**(크레딧·시간 보호)

## 실행

```bash
# 1) 프롬프트만 확인 (키·크레딧 불필요)
node .claude/skills/lumenstate-product-variation/scripts/generate.mjs --product 17 --dry-run

# 2) 실제 생성
node .claude/skills/lumenstate-product-variation/scripts/generate.mjs \
  --product 17 --glass opaline --hardware patina-brass --count 4
# → generated/lumenstate/17/17-angle-offdim.png … 4장
```

실패 모드(의도적 fail-fast): `GEMINI_API_KEY` 미설정 · 패키지 미설치 · 존재하지 않는 id · 허용되지 않는 재료 값 → 안내 메시지 후 즉시 종료.
