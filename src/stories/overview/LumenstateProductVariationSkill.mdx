# Lumenstate — Product Variation Skill

> 제품 상세페이지의 **베리에이션 컷(대표 사진 아래 갤러리 썸네일)** 을 자동 생성하는 도메인 한정 스킬.
> 위치: `.claude/skills/lumenstate-product-variation/`

---

## 한 줄 정의

상세페이지에 등록된 **대표 이미지 `src/assets/product/{id}.png`(낮·소등)** 를 Gemini(Nano Banana 2)에 **레퍼런스로 전달하는 image-to-image** 방식으로, 대표 사진과 **100% 동일한 제품**을 유지한 채 그 제품을 **실제 인테리어 공간**(`product.type` 매핑)에 배치하고 **각도 · 줌 · 디테일 크롭**으로 변주해 N장 생성한다. off↔on 점등 연속성을 베이스로 깔고, 컷들은 **항상 병렬로** 생성한다.

> **두 축을 동시에 만족시킨다:** ① 제품은 텍스트로 그리지 않고 레퍼런스 PNG 로 고정(그래야 대표 사진과 같은 제품) ② 단색 스튜디오가 아니라 인테리어에 배치(그래야 실제 배치 감각). 하나라도 어기면 — 텍스트로 그리면 다른 제품, 단색 배경이면 카탈로그 컷 — 갤러리 의도가 깨진다.

---

## 언제 발동하는가

**Lumenstate 제품 베리에이션 / 상세 갤러리 컷 / 베리에이션 썸네일 생성**, 또는 products.js 제품을 대표 사진 기준으로 변주해 달라는 요청. `Lumenstate`, `베리에이션`, `제품 컷`, `Nano Banana`, `gemini image` 키워드가 트리거. **도메인 전용**이며 임의 이미지 생성과 구분된다.

## 핵심 원칙

1. **레퍼런스 이미지가 제품을 정의한다.** `src/assets/product/{id}.png`(낮·소등 대표샷)를 입력하고 형태·비율·색·재질·마감을 **100% 동일**하게 유지. 재설계·부품 추가 금지.
2. **그 제품을 실제 인테리어에 배치한다.** 단색 스튜디오가 아니라 `product.type` 에 맞는 공간(CEILING→거실/다이닝 천장 매입 등)에 실제 설치된 모습으로.
3. **변하는 건 카메라·점등뿐.** 각도 · 줌 · 디테일 크롭 + off/on. 제품 자체는 모든 컷에서 같다.
4. **빛의 연속성.** `{id}.png`(소등·밝은 공간) ↔ `{id}-1.png`(점등·어두운 공간 글로우)를 따라 off↔on 배치.
5. **병렬 전제.** 컷은 독립적 → `Promise.all` 로 항상 동시 생성(한 컷 실패해도 나머지 계속).

## 변주 지시 (텍스트 5블록)

각 컷의 `contents` = `[ 레퍼런스 이미지(inlineData), 변주 지시 텍스트 ]`.

```
1) 고정 지시  레퍼런스 제품을 100% 동일하게 유지(재설계 금지). 이 제품을 아래 인테리어에 실제 설치된 모습으로 배치.
2) Setting    product.type → 인테리어 공간 (space-mapping).
3) Variation  컷별 각도 · 줌 · 디테일 크롭.
4) Lighting   off / on. on 이면 kelvin 색온도.
5) Style      Lumenstate 인테리어 톤(warm minimal, ambient warmth, flat & quiet).
```

> 1번(고정 지시)·5번(Style)은 공유 상수. 변하는 건 3번(각도/줌/크롭)·4번(점등)이고, 2번(Setting)은 제품별 고정.
> 매크로 크롭은 "collage/grid 금지"를 명시하지 않으면 4분할 콜라주가 나온다 — "ONE continuous frame" 유지.

## Gemini API · 환경 설정

- 모델: `gemini-3.1-flash-image-preview` (Nano Banana 2)
- `ai.models.generateContent({ model, contents: [imagePart, textPart], config })`
- `config.imageConfig`: `aspectRatio: '1:1'`, `imageSize: '1K'` (1K ≈ 1024²; 프리뷰는 imageSize 무시 — js-genai #1461)
- ⚠️ `outputMimeType` 은 **Developer API 미지원**(Enterprise 전용) → 넣지 말 것. 모델이 JPEG 를 반환하므로 저장 시 `inlineData.mimeType` 으로 확장자를 맞춘다.
- 반환: `response.candidates[0].content.parts[].inlineData.{data, mimeType}` (base64).

### API 키 가이드

1. [aistudio.google.com](https://aistudio.google.com) 발급
2. 루트 `.env.local` 에 `GEMINI_API_KEY=발급키`
3. `.gitignore` 의 `*.local` 이 이미 제외
4. 미설정 시 **명확한 에러로 즉시 종료**

## 실행

```bash
# 프롬프트만 확인 (키·크레딧 불필요)
node .claude/skills/lumenstate-product-variation/scripts/generate.mjs --product 1 --dry-run

# 실제 생성 (4컷 병렬, 1.png 를 레퍼런스로 인테리어에 배치)
node .claude/skills/lumenstate-product-variation/scripts/generate.mjs --product 1 --count 4
# → generated/lumenstate/1/1-room-3q-day.jpg … 4장
```

실패 모드(fail-fast): `GEMINI_API_KEY` 미설정 · 패키지 미설치 · 존재하지 않는 id · `src/assets/product/{id}.png` 부재 → 안내 후 즉시 종료.
